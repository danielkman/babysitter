# Project Deployment — Uninstall Instructions

## Step 1: Interview the User

Ask the user what to remove:

1. **Processes and skills only** — Remove babysitter deployment processes/skills/agents but keep all generated infrastructure config (Terraform, K8s, Dockerfiles, CI/CD pipelines)
2. **Everything** — Remove processes, skills, agents, AND all generated deployment configuration files
3. **Selective** — Let the user choose which components to remove

**Warning**: Removing infrastructure configuration files (Terraform, Helm charts, CI/CD pipelines) may break existing deployments. Confirm with the user before proceeding.

## Step 2: Remove Processes, Skills, and Agents

```bash
rm -rf .a5c/processes/deployment
rm -rf .a5c/skills/deployment
rm -rf .a5c/agents/deployment
```

## Step 3: Remove Generated Configuration (if user chose "Everything" or "Selective")

Only remove files that were generated by this plugin. Check git history to identify plugin-generated files:

```bash
# These are examples — only remove what was actually generated during install
# Terraform
rm -rf terraform/

# Kubernetes/Helm
rm -rf k8s/
rm -rf helm/

# CI/CD (only deployment-specific workflows, not all workflows)
rm -f .github/workflows/deploy.yml
rm -f .github/workflows/deploy-staging.yml
rm -f .github/workflows/deploy-production.yml

# Platform config (only if generated)
rm -f vercel.json
rm -f wrangler.toml
rm -f serverless.yml
```

**Do not remove** `Dockerfile` or `docker-compose.yml` if they existed before plugin install.

## Step 4: Remove from Registry

```bash
babysitter plugin:remove-from-registry --plugin-name project-deployment --project --json
```

## Notes

- Live infrastructure (cloud resources, running containers, DNS records) is NOT affected by uninstalling the plugin
- To tear down actual cloud infrastructure, run `terraform destroy` or delete resources via the cloud console before removing config files
- CI/CD pipelines in progress will complete but no new deployments will trigger after workflow files are removed
