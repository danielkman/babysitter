## Single-Channel Communication

- For any given task, pick one channel and stay there: the PR conversation, the issue thread, or a chat thread — not all three.
- Do not cross-post the same update. Pointing once ("followed up in #123") is fine; duplicating is noise.
- If the conversation must migrate (e.g. from an issue to a PR once code exists), leave a single pointer comment in the old channel and continue in the new one.
