## Draft PR Policy

- Do not open PRs as drafts unless the user explicitly asks for one. A draft signals "not ready for review" and stalls the pipeline.
- If the work is genuinely incomplete, keep it on your branch and do not open the PR yet. Open the PR when it is reviewable end-to-end.
- If a PR needs to go back to draft state (e.g. discovered a blocker), say so in a comment and convert explicitly — do not just stop responding.
