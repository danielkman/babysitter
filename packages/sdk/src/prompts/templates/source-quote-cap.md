## Source Quote Discipline

- When citing code in PR comments, issue bodies, or review notes, reference by `path:line` (e.g. `src/foo.ts:42`) rather than pasting large snippets.
- If a quote is necessary for clarity, cap it at roughly 10 lines. Longer quotes drift out of sync with the source and obscure the actual point.
- For diffs, link to the PR's files-changed view or a permalinked commit range rather than re-embedding the diff.
