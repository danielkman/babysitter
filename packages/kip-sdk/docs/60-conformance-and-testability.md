# Conformance & testability

> The conformance invariant catalog (INV-1..INV-19, INV-A1..INV-A14) presented as a structured TEST PLAN: for each invariant, what it asserts and what a violating build looks like.

**Source: SPEC §8.4 (3119–3449).** Convergence claims trace to [synchronization & convergence](./24-synchronization-and-convergence.md) (§4b/§7); substrate / retention to [git substrate](./22-git-substrate.md) (§3); bitemporal axes to [temporality & bitemporality](./23-temporality-and-bitemporality.md) (§4); active-layer invariants to [active knowledge overview](./30-active-knowledge-overview.md) (§5b).

RFC-2119 keywords are normative. This doc adds **no** requirements beyond §8.4; it restructures the spec's invariant list as a test plan.

---

## 0. Strategy: determinism is the testing strategy

The whole suite leans on one lever: **author-stamped HLC, fixed reducer seeds, fixed `replicaId`s, content-addressed everything, and `rxFrom` excluded from every convergent path** make every assertion reproducible. The standard test shapes are:

- **Random-permutation fold** — fold random permutations / partitions of the full fact multiset and assert **byte-identical** output (the real test of a CRDT fold; replaces any binary-triple ACI test).
- **Random-order / random-partition replay** — deliver the same facts to N replicas in different orders / partitions and assert equal `/heads`.
- **Adversarial perturbation** — perturb `rxFrom`, commit grouping, the simulated receiver clock, and key-registration arrival order, and assert outputs are **unchanged**.

### 0.1 Byte-identity vs recall-equivalence — the accelerator boundary (§5.3)

Two assertion classes, and they MUST NOT be confused:

| Class | Asserts | Applies to |
|---|---|---|
| **Byte-identity** | exact byte equality of `/heads`, projections, segments, regenerated DAG | **deterministic** projections, all of `proj`, the substrate, the convergence core |
| **Recall-equivalent** | a recall@k threshold (statistical equivalence), NOT byte equality | **accelerator** projections only — ANN / embeddings (§5.3, INV-5, M-7) |

The accelerator (ANN) tests are **recall-threshold, not byte-equality**. Everything else is byte-equality.

## 1. Core convergence & projection (INV-1, INV-2, INV-3)

These are the load-bearing correctness invariants for [convergence](./24-synchronization-and-convergence.md).

- **INV-1 — proj determinism + replica-local-input independence.** *Asserts:* `proj(factSet)` ⇒ byte-identical `/heads` regardless of delivery order, batching, **`rxFrom`, commit-order, the receiver's physical clock, the local key-log sync state, or wall-clock-at-read** (C2-1, C3-1). A dedicated test perturbs each fact's `rxFrom` / ingest order, commit grouping, the simulated receiver clock, and key-registration arrival order, and asserts `/heads` is **unchanged** — proving `proj` reads no replica-local quantity. `/heads` is advisory; `fsck` asserts committed `/heads == proj(facts)`. *Class:* byte-identity. *Violation:* any replica-local quantity leaking into `proj` so perturbation changes `/heads`.
- **INV-2 — convergence / SEC over the SIGNATURE-VALID admitted set (non-vacuous, C3-1).** *Asserts:* the admitted set is **exactly the signature-valid received set** (membership is signature-only, §3.2), so for random fact-set partitions delivered in random orders to N replicas — **including key-authorization, `revoke-key`, and backdated / anachronistic facts crossing author-HLC boundaries** — once the same facts have been *received* (not gate-filtered differently), all replicas reach byte-identical `/heads` and byte-identical **deterministic** projections (modulo the bounded excision-propagation window, §4b.4). The suite includes the C2-1 counterexample (a fact straddling a revocation, merged late on one replica) **and** C3-1 stress (skewed receiver clocks, key-registration facts arriving *after* the data facts they authorize, long offline partitions) and asserts **equal** heads — catching **membership** divergence, not only value divergence. *Class:* byte-identity (accelerator projections excluded, see INV-5). *Violation:* any membership or value divergence across the permutations.
- **INV-3 — reducer determinism + `orderKey` totality.** *Asserts:* every registered `CellReducer`, folded over **random permutations of the full fact multiset for a cell**, yields byte-identical segments; every reducer's final tiebreak terminates in `orderKey`. Additionally **no two distinct admitted facts share an `orderKey`** (totality — checked by asserting the canonical payload covers `publicKeyFingerprint`, `replicaId`, `v`, M2-1). *Class:* byte-identity. *Violation:* a non-total or non-deterministic reducer, or an `orderKey` collision between distinct facts.

## 2. Bitemporal & belief axes (INV-4, INV-11)

See [temporality & bitemporality](./23-temporality-and-bitemporality.md) for the underlying valid/transaction-time model.

- **INV-4 — belief-consistency (per-replica AUDIT; renamed from "bitemporal soundness", M2-4).** *Asserts:* projected segments of any cell are **non-overlapping at every rxTime slice; gaps read as `Unknown`**. `asOf({txTime, believer=R})` agrees with a **per-replica** reference oracle built from R's ingest order. This is a **belief-audit** invariant, **explicitly NOT a convergence claim** (txTime is replica-local). *Class:* per-replica audit (not cross-replica byte-identity). *Violation:* overlapping segments at a slice, or `asOf(txTime)` disagreeing with R's oracle.
- **INV-11 — validTime convergence (closes M2-4).** *Asserts:* equal admitted sets ⇒ **byte-identical** `asOf({validTime})` on all replicas. A dedicated test perturbs `rxFrom` / ingest order and asserts the validTime answer is unchanged — the convergent counterpart to INV-4. *Class:* byte-identity. *Violation:* any regression that leaks `rxFrom` into the validTime path.

## 3. Projection rebuildability & idempotence (INV-5, INV-7, INV-8)

- **INV-5 — projection rebuildability (scoped).** *Asserts:* dropping and rebuilding all **deterministic** projections yields byte-identical results. **Accelerator** projections (ANN / embeddings) rebuild to **recall-equivalent** (a recall@k threshold test), **NOT** byte-identical (§5.3, M-7). *Class:* byte-identity (deterministic) + recall-equivalent (accelerator) — this invariant is where the boundary is drawn. *Violation:* a deterministic projection that does not rebuild byte-identically, or an accelerator held to byte-identity it cannot meet.
- **INV-7 — idempotent ingestion.** *Asserts:* re-ingesting any fact set is a no-op — CID dedup holds because the author-stamped, signed HLC is part of the CID (M-4), so the same logical fact has one CID on all replicas (no double-count under `pncounter`). *Class:* byte-identity. *Violation:* a re-ingest that double-counts or duplicates valid-time intervals.
- **INV-8 — upcaster soundness (terminates, never invents).** *Asserts:* `upcast(v_old → v_n)` **terminates with a typed result** (`value | quarantine`) for every historical and future fact version; unknown versions pass through as **opaque-quarantined**; it **never throws and never invents missing required data** (M-8, honoring the no-fallback rule N5). *Class:* termination / typed-result. *Violation:* an upcaster that throws, hangs, or fabricates required data (the corrected v1 "total / never quarantines" claim).

## 4. GC, excision & regenerated DAG (INV-9, INV-12)

- **INV-9 — GC / excision safety.** *Asserts:* gc / repack / rollup **never** change query results for any non-excised `asOf`; excision changes results **only** for the excised CID (returning an `"excised"` placeholder) and re-folds dependent heads. *Class:* result-stability. *Violation:* gc altering a non-excised result, or excision residue surviving in `/heads`.
- **INV-12 — concurrent-excision pin / as-of convergence + BYTE-IDENTICAL regenerated DAG (closes m2-4, C2-3, M3-3).** *Asserts:* after **concurrent** excision on different replicas, `asOf` and durable pins (`factSetDigest` + author-HLC frontier) resolve **identically** across replicas, **independent of commit-CID divergence**, and the **regenerated** commit DAG is **byte-identical** given the equal remaining ordered set. The suite excises F1 on A and F2 on B concurrently, syncs, and asserts equal `/heads`, equal pin resolution, and byte-identical regenerated DAG. Byte-identity holds **iff** every regenerated commit field is set-derived: deterministic `orderKey`-based commit boundaries, commit timestamp = the batch's max author-HLC `wall`, **fixed sentinel committer**, **unsigned** DAG (M3-3). **Cross-OS / cross-TZ byte recipe (M4-3):** runs the regenerator on a `+0200`-local replica and a `+0000` replica with mismatched `core.autocrlf` and locale, asserting byte-identical commit objects — catching any leak of timezone offset (must be fixed `+0000`, integer-seconds `floor(wall/1000)`), commit-message line endings (LF-only), `encoding` header (absent / UTF-8), `gpgsig` header (absent), or author≠committer sentinel (single identical sentinel). *Class:* byte-identity. *Violation:* a build that stamps "now", uses the regenerator's identity/key, reuses transport batch boundaries, or leaks any TZ/EOL/header/sentinel detail.

## 5. Trust, gate/proj separation & authority (INV-6, INV-10)

These mechanize the signature-only gate vs set-pure proj demotion (§3.2 / §3.6 / §8.1). See [convergence §6](./24-synchronization-and-convergence.md).

- **INV-6 — gate / proj separation + verification (signature-only gate, C3-1, M3-4).** *Asserts:* the **ingest gate** rejects **only** signature-invalid or malformed facts (the sole byte-pure predicate) — **identically on every replica** (membership convergence). **Unregistered-key, out-of-namespace, revoked, AND author-HLC-anachronistic facts are NOT rejected at ingest**; they are **demoted-untrusted / quarantined inside `proj` keyed on author-HLC** — surfaced, never silently trusted or dropped, re-evaluated monotonically. A dedicated test feeds a signature-valid fact whose key is *unregistered at ingest time*, asserts it is **admitted**, then delivers the key-registration fact and asserts it becomes **trusted** on re-fold. Excision preserves verifiability of remaining **fact** signatures and re-folds `/heads` so no excised residue survives. **Commit signatures are NOT a trust anchor and are NOT checked** (M2-2); the regenerated DAG is unsigned with a fixed sentinel committer and commit-author ≠ fact-author (allowed, M3-3). *Class:* membership-identity + monotone re-fold. *Violation:* a gate that rejects on anything but signature/well-formedness, or a build that silently trusts/drops a demotable fact.
- **INV-10 — authority chain (author-HLC keyed).** *Asserts:* every fact whose author key chains (**at the fact's author-HLC**) to its tenant genesis root for the target namespace projects **trusted**; one that does not is **demoted-untrusted by `proj`** (not rejected at ingest). A key-add or excision/revocation whose *authoring* key lacks the scoped capability is itself demoted-untrusted by the same author-HLC rule. *Class:* set-pure trust decision. *Violation:* an authority decision that reads `rxFrom`/receiver clock, or rejects-at-gate instead of demoting.

## 6. Substrate convergence reachability & pins (INV-13, INV-14)

- **INV-13 — signature-valid ⇒ eventually-admitted-on-receipt (non-vacuous substrate convergence, C3-1/C3-2/M3-4).** *Asserts:* every **signature-valid** fact offered to any replica is **admitted by every replica that receives it, upon receipt** (eventual-on-receipt — *not* "within bounded time," unstatable in a partition-tolerant async model), **regardless** of clock skew, partition length, or whether its signing key's registration has arrived yet — and its *logical membership* is **never dropped** (its bytes MAY be retention-evicted per §3.5a if non-durable, but a peer still serves the blob on demand). The suite delivers signature-valid facts under adversarial clock skew, multi-day offline partitions, and data-before-key-registration ordering, and asserts each fact is present in every replica's admitted set after sync (a quarantined/untrusted fact still counts as *admitted*). *Why it matters:* this is the conformance-level proof that the SEC antecedent (equal admitted sets, modulo §3.5a retention) is **actually reachable, not vacuous** — only statable truthfully because admission is signature-only. *Class:* membership presence. *Violation:* any signature-valid fact rejected/dropped on receipt under skew/partition/key-ordering.
- **INV-14 — pin completeness & stability (M3-2 / m3-3).** *Asserts:* a pin resolves to the deterministically-selected subset `{ f ∈ S : f.authorHlc ≤ frontier[f.replicaId] }` (replicaId absent ⇒ excluded). The suite takes a pin, delivers a **late-arriving sub-frontier fact** to one replica only, and asserts the lagging replica reports **`pin-incomplete`** (never a silent partial digest mismatch); after the fact syncs everywhere, every replica reports **`pin-complete`** with the **identical** `factSetDigest`. A causal anachronism or key-registration race MUST NOT change the pinned subset's digest once complete. Completeness is decided by the **per-replica HLC-counter contiguity rule** (unbroken `(wall,counter)` chain up to `frontier[R]`, m4-1). *Class:* byte-identity of digest + local completeness decision. *Violation:* a build that cannot locally decide completeness, or that returns a silent partial-digest mismatch.

## 7. Causal plausibility & anti-backdating (INV-15, INV-16, INV-19)

The set-pure causal-trust core (§4b.1, §3.6) — the part most prone to subtle convergence holes. See [convergence §1.3](./24-synchronization-and-convergence.md).

- **INV-15 — `causedBy` well-formedness (set-pure, M4-2).** *Asserts:* a fact whose any `causedBy` parent resolved in `S` has author-HLC **>** the fact's own author-HLC is demoted `untrusted-malformed`; a `causedBy` cycle is demoted-malformed; an unresolved (not-yet-in-`S`) `causedBy` parent leaves the fact **pending** (not trusted), never silently trusted. The suite feeds (a) a forward `causedBy` edge, (b) a `causedBy` cycle, (c) a dangling `causedBy`, and asserts each **demotes/pends** rather than projecting trusted, on every replica identically (acyclicity guarantees closure-walk termination). *Class:* set-pure demotion, byte-identical. *Violation:* any of (a)/(b)/(c) projecting trusted.
- **INV-16 — per-key anti-backdating from INVOLUNTARY footprint, GATED ON CHAIN COMPLETENESS (C4-2 + C5-1, m5-5).** *Asserts:* a signature-valid fact `F` from key `K` projects **trusted** only over a complete gap-free `(wall,counter)` chain of `K` up to `F` (else `pending`), and once complete is demoted `untrusted-anachronistic` iff `S` holds a higher-author-HLC, non-ancestor fact from the **same** `K` **IN THE COMPLETE DURABLE CHAIN** — **independent of whether `F` declares `causedBy`**. Sub-cases:
  - **(C4-2)** author a registered, non-revoked key's higher-stamped fact, then a `causedBy`-LESS backdated fact from the same key → the backdate is **demoted** (NOT trusted).
  - **(genuine residual)** a lone self-dated fact from a key that **genuinely emitted nothing higher in its chain** → projects **trusted** (the acknowledged acceptable residual, §3.6).
  - **(C5-1 eviction case, m5-5)** the "no higher same-key fact" antecedent is "no higher same-key fact **IN THE COMPLETE DURABLE CHAIN**." A higher same-key fact that exists but is **missing / evicted / unreplicated** on this replica ⇒ the backdate must project **`pending`** (chain gap), **NOT trusted**, and flip to **demoted** once the higher fact (and intervening chain) arrives.
  - the decision is byte-identical across replicas with complete chains.
  *Class:* set-pure trust, byte-identical on complete chains. *Violation:* projecting the eviction-route backdate **trusted** (the C5-1 vulnerability).
- **INV-19 — anti-backdating preserved under eviction & partial replication (C5-1; holds under cap-bounded retention, M6-1/m6-2).** *Asserts:* per-key trust is **monotone and eviction-safe**. The suite (1) authors `K`'s higher honest fact `F'` and a lower backdate `F` from the same registered `K`; (2) on a victim replica induces eviction / non-replication of `F'` (whether `F'` is an unregistered-key fact **or** a registered chain link evicted **past `keyChainDurableCapBytes`** — both reproducible under M6-1); (3) asserts the victim projects `F` **`pending`** (chain gap), **never trusted**; (4) re-fetches/delivers `F'` (and the intervening chain) and asserts `F` flips to **demoted** (`untrusted-anachronistic`), **at most once and never reversing** — **non-reversal holds under cap-bounded retention because the completed-chain frontier is pinned** (a link backing a non-`pending` dependent is not re-evictable, m6-2). The contiguity gap is detected **per-`(replicaId,key)`** and MUST be caught even when `F'` was authored on a **different replica** than the dependent's other chain links (m6-1). *Class:* monotone set-pure trust under retention. *Violation:* any eviction / selective non-replication flipping a same-key backdate from `pending`/`demoted` to `trusted`, or a `pending → demoted/trusted` transition that reverses under cap-bounded retention.

## 8. Revocation (INV-17)

- **INV-17 — revocation intent & honest-concurrent surfacing (M4-1).** *Asserts:* `ordinary-cutoff` revocation demotes **only** facts with author-HLC ≥ `effectiveFrom` and **preserves** honest concurrent pre-`effectiveFrom` facts as trusted; `causal-cutoff` additionally demotes non-ancestor facts but surfaces honest-concurrent casualties with the **distinct** status `kip:revoked-concurrent` (queryable via `recall`/`provenanceOf`), never the generic `untrusted` bucket. The suite authors honest concurrent pre-`T` facts during a revocation's propagation window and asserts: under `ordinary-cutoff` they stay trusted; under `causal-cutoff` they are demoted **and** carry `kip:revoked-concurrent`. It further asserts a **`re-attest` fact signed by a currently-trusted, non-revoked key** (`reAttests` = the demoted fact's CID) **restores** the honest content to a trusted head under the new key's provenance (original demoted fact stays verifiable in history), while a `resolve`-scoped `supersede` over the same `inputCids` does **NOT** restore it (wrong primitive — there is no conflict to adjudicate). *Class:* set-pure trust + distinct status surfacing. *Violation:* honest pre-`T` facts demoted under `ordinary-cutoff`, casualties bucketed as generic `untrusted`, or `re-attest`/`supersede` confused.

## 9. Admission-control, retention & partial-replication SEC (INV-18)

The single largest invariant — mechanizes the §4b.4 partial-replication corollary and the C5-1 hole closure. See [convergence §4.2–4.3](./24-synchronization-and-convergence.md) and [git substrate §3.5a](./22-git-substrate.md).

- **INV-18 — admission-control / retention resource bound + partial-replication SEC (C4-1, m5-1/m5-2, C5-1).** *Asserts:* an unregistered key's flood of distinct signature-valid facts is admitted **logically** (INV-13 still holds) but computes `RetentionClass = quarantined-ttl` and is eviction-eligible under the per-key cap/TTL **and the global aggregate budget**; a registered key's **trusted** facts compute `durable` (never evicted) and its chain-anchoring facts compute `key-chain-durable` (preferentially retained up to `keyChainDurableCapBytes`, cap-bounded with on-demand re-fetch, M6-1). The suite floods from a fresh unregistered key and asserts:
  - **(a)** trusted heads are **unchanged** (the flood projects `quarantined`, affecting no cell);
  - **(b)** durable bytes are bounded **under the default retention policy** (the named antecedent, m5-2 — a permissive opt-out replica is *not* claimed bounded; **INV-18(b) is NOT an unconditional guarantee**). It additionally floods from **`N` distinct fresh unregistered keys** and asserts the **aggregate** `quarantined-ttl` pool never exceeds the global `quarantinePoolBytes` budget (m5-1) — a build whose only ceiling is per-key (so `N` keys ⇒ `N×` bytes) **fails**;
  - **(c)** **per-shared-subset SEC + the C5-1 regression test** — two replicas with different evicted subsets agree on `proj` over the *complete-durable* part of `S_A ∩ S_B`, and fully on the `durable`/`key-chain-durable` subset (§4b.4 corollary); **and** evicting a **pre-registration same-key** fact does **NOT** flip a later same-key fact from demoted to trusted — it instead yields a chain gap so the later fact projects **`pending`** (the C5-1 hole; a flip-to-trusted build **fails**);
  - **(d)** `RetentionClass` is computed identically on every replica (set-pure), and a **registered key's chain is `key-chain-durable` — preferentially retained UP TO `keyChainDurableCapBytes`, NOT never-evicted (M6-1)**: (d1) within the cap, links are retained and honest facts project trusted without re-fetch; (d2) **past the cap, the oldest non-load-bearing links MAY be evicted, and an evicted link forces dependent same-key facts `pending` (never trusted) until re-fetch** — a build that flips a dependent to trusted across a cap-evicted gap (C5-1) **fails**, and a build whose registered-key durable pool is **unbounded** (no cap enforced) **also fails**; (d3) a link backing a currently non-`pending` dependent is **pinned** (completed-chain frontier, m6-2) so INV-19's non-reversal holds. Contiguity is per-`(replicaId,key)`; completeness is the union over `K`'s replicas; the monotonicity demotion is key-wide (m6-1).
  *Class:* set-pure retention computation + per-shared-subset byte-identity (durable/key-chain-durable subset). *Overall violation:* a build that lets a non-durable eviction change a trusted head, has no aggregate retention bound, or leaves the registered-key durable pool unbounded — **fails**.

## 10. Active-knowledge conformance invariants (INV-A1..INV-A14, §5b)

These ship with the suite too. **INV-A1 is the prose load-bearing rule; INV-A2..INV-A14 are its §8.4-style testable counterparts**, each with an adversarial recipe mirroring INV-1..19 — and **all cover their area WITHOUT touching the convergence core**. See [active knowledge overview](./30-active-knowledge-overview.md).

| Invariant | Area | Asserts (summary) | Violating build |
|---|---|---|---|
| **INV-A1** | microagents-are-clients (load-bearing) | **No** active-layer path (`registerFunctionality`/`runContextualQuery`/`runAcquisition`/`learn`) mutates `/heads` except by **appending a signed fact**; run against a harness whose only mutation primitive is `assertFact`; every state change attributable to a signed fact authored by the orchestrator. | a bound functionality, encode/decode/learner, or Miner/Ingestor that writes the graph directly. |
| **INV-A2** | compile-determinism + DAG order | Two replicas compiling the **same** `ContextualQuery` at the **same** `asOf` produce **byte-identical** `Segment` sets (steps + `deps` + ordered `alternatives`), under perturbed sync state / key-log order / wall clock; a branching `deps` DAG yields a **byte-identical TOPOLOGICAL order** (ties: ascending `steps[]` index then §3.4 tiebreak); a **cyclic** or out-of-range `deps` is **rejected at compile**. | compile reading replica-local sync state or a wall clock; replica-dependent topo order; admitting a cyclic `deps`. |
| **INV-A3** | dispatch no-fallback (N5) | For one hop each, inject (a) non-zero `exitCode`, (b) `outputSchema`-invalid output, (c) timeout > `runtime.timeout`, (d) unsatisfied `requires`/`condition` guard, (e) a **claim-8 `constraint` the seed violates** → in **every** case **zero facts authored**, cell reads `Unknown`; (d)/(e) differ from a dispatch failure **only in recorded provenance**. | fabricating a plausible output; surfacing a pending guard or constraint-violating seed as a trusted value; dispatching on a non-compliant seed. |
| **INV-A4** | learner replica-fold | Commit a `kip:learn` fact on A, then **drop & rebuild `proj`** on B that has the fact but **lacks the models** → B's learned heads **byte-identical** to A's (proj NEVER re-runs the loop). | `proj` re-invoking any accelerator-class agent (so model-less B diverges/errors). |
| **INV-A5** | budget termination (3 axes) + exhausted marker | Three sub-cases forcing `loss ≥ threshold` forever — (a) tiny `maxIterations`, (b) tiny `maxWallMs` + hung decode, (c) tiny `maxInvocations` + high fan-out → each: `converged()` = `"exhausted"`, **no `accept` fact**, a single signed `kip:learn-exhausted` marker, cells stay `Unknown`. | `converged` ignoring `maxWallMs` or `maxInvocations` (so (b)/(c) loop unbounded). |
| **INV-A6** | hop idempotence | Re-run an **identical** hop on identical input → second run is a `factCID`-dedup **no-op** (INV-7 linkage); identical instance resolves to the **same** namespaced EID (node-merge). | re-minting distinct facts or a second EID for the identical hop. |
| **INV-A7** | multi-segment & multi-realizer typed choice (N5) | Two segments satisfying a query (differing `weight`), AND two `FunctionalityBinding` realizers on one `(edgeKind,sourceKind,targetKind)` hop → `runContextualQuery` surfaces a **typed choice** (`Segment.alternatives` non-empty, ordered by `weight` then §3.4 tiebreak), executing **nothing** until the caller chooses (mirrors `kip:conflict`); a `NaN`/`±Infinity` `weight` or `range`/`cmp` comparand is **rejected at registration**. | auto-executing the top-weighted (or any `tags`/hash-tiebroken) segment/realizer; admitting a `NaN`-weight binding. |
| **INV-A8** | answer-graph = derived_from projection | Multi-hop query → returned `AnswerGraph` is **byte-identical** to `proj`'s projection of the `derived_from` subgraph at the recorded `asOf`; **every** `result`/`intermediate` EID is reachable from `seed` along `derived_from`. | an `AnswerGraph` node with no `derived_from` path back to `seed`; authoring the graph as a separate artifact instead of reading it back from facts. |
| **INV-A9** | proj-totality over §5b cells + loss-exclusion | Random-permutation fold of full multisets for `kip:learn`, `kip:learn-exhausted`, microagent-registration, `derived_from`, `same_as` cells → byte-identical, tiebreak terminating in `orderKey`. Two competing `kip:learn` (same key, different accepted set, **lower-loss given LOWER `orderKey`**) → winner is **`orderKey`-max** (or `kip:conflict`), **never** the lower-loss one. Two `kip:learn` same key + same accepted `AssertInput[]` + **different loss** → fold as **one no-op** (NOT `kip:conflict`) — loss canonicalized out of the dedup key. | routing recorded loss into the reducer/tiebreak; or surfacing `kip:conflict` on a same-set/different-loss pair. |
| **INV-A10** | acquisition family lifecycle + ordering/kind-preservation + divergent-registration conflict | (a) `AcquisitionResult.proposed` lands **quarantined-until-trusted** (only trusted via the ordinary §8.1 path, never trusted-on-import); (b) each `sameAs` → exactly one signed `same_as` fact, contradicting merge → `kip:conflict`; (c) Discoverer traversal **terminates within its declared bound**; (d) a family microagent writing `/heads` directly **fails** (INV-A1 parity); (e) mixed `proposed` batch preserves **kind** (`AssertInput`→`assert`, `RetractInput`→`retract`, no coercion); (f) returned `{facts}` is **exactly** `proposed` order then `sameAs` order; two registrations of same `(name,version)` with divergent manifests → cell reads `CONFLICTED` (not LWW-overwrite). | coercing `retract`↔`assert`; reordering the returned `FactId[]`; LWW-overwriting an incompatible descriptor; sourceless Miner writing `/heads`. |

## 11. Reproducibility footing

Determinism (author-stamped HLC, fixed reducer seeds, fixed `replicaId`s, content-addressed everything, **`rxFrom` excluded from every convergent path**) makes all of these reproducible. The **accelerator (ANN) tests are recall-threshold, not byte-equality** — the one place §5.3's boundary is exercised (INV-5, INV-2's exclusion). Everything else asserts byte-identity.
